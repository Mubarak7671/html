Differences b/w Positions

static : Default position; element stays in normal flow and can’t be moved. relative : Moves element slightly from its original spot without affecting others. absolute : Removed from normal flow; positioned relative to nearest positioned ancestor. fixed : Stays fixed on the screen even when scrolling (relative to viewport). sticky : Acts relative until a scroll point, then sticks like fixed.

Defference b/w inline,inline-block,block and flex display properties

Inline

  • Elements appear in same line ,with out line breaks
  • Ex: span,a,stonge,margin etc... type tags
  • We can't set weidth and hight in inline

inline-block

  • Behaves like inline (stays in line) but allows width and height adjustments.
  • useful for placing multiple boxes side by side with control over size.

Block

  • Always starts on a new line and stretches to fill available width. For ex:We fill in content in "p tag" after closing "/p tag" forther content enter into next line
  • We can set width, height, margin, and padding freely for block elements. Common examples: div, p, h1,h6.li

Flex

  • Turns a container into a flexible layout; items align in row or column. control how a flex items grow,shrink,and sets its initial size inside a flex container
  • helps create responsive, aligned, and spaced designs easily.

Explanation of Z-Index and Stacking context

z-index controls which element appears on top when elements overlap. Higher z-index → element comes forward (on top). Lower z-index → element goes behind.

Difference b/w transition and animation

The transition property makes CSS changes smooth and animated.

  • When you move your mouse over the box (hover), it slowly changes its color and size in 1 second. Without transition, it would change instantly.

The animation property makes elements move or change automatically — you don’t need to hover or click. It uses keyframes to define how the element should change step by step.
